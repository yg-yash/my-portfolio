Beedwala - Mobile App (React Native with Express.js and MongoDB)

About the Project

The **Beedwala** app is a mobile platform built using **React Native** with a **backend powered by Express.js** and a **MongoDB database**. The app serves as an online marketplace for beekeepers to buy and sell honey, beekeeping supplies, and other related products. Users can browse through various products, make purchases, and track their orders seamlessly through the app.

Key Features

  • Product Listing: The app displays products related to beekeeping, including honey, equipment, and supplies, making it easier for users to browse and buy items.
  • Cart and Checkout: Users can add items to the cart and proceed with secure checkout using the payment gateway integrated into the backend.
  • User Authentication: The app allows users to sign up, log in, and access their orders and personal information securely.
  • Order Tracking: Beekeepers can track their orders and see their delivery status in real-time.

Technologies Used

The mobile app is built using **React Native**. The backend is developed with **Express.js**, and the data is stored in a **MongoDB** database. The app uses **JWT** for user authentication and **Stripe API** for payment processing.

Outcome

The **Beedwala** app enables beekeepers to easily buy and sell products, while providing a simple and intuitive shopping experience. The app's efficient backend with **Express.js** and **MongoDB** ensures smooth transactions and quick data handling, while **React Native** allows for a native-like experience on both iOS and Android devices.

ooDzee - Food Order and Delivery App

Introduction

**ooDzee** is a React Native-based mobile application designed to provide a seamless user experience for food ordering and delivery. The app features an elegant UI with intuitive navigation and integrates Google Maps to display restaurant locations. While this is a UI-focused project without live API integration, it lays a strong foundation for a fully functional app that can handle real-time data and orders in the future.

Project Steps

  • Setting Up the Project Structure - The first step was to set up the React Native project using the latest version of React Native. We structured the project to incorporate popular navigation libraries like React Navigation, and Redux for managing the state across the app. This provided a solid foundation for future development.
  • UI Design and Layout - A primary focus of the project was the user interface. We designed multiple screens, including the home page, restaurant listing, menu display, and map integration, using React Native Elements and custom UI components. The goal was to create a visually appealing and easy-to-navigate experience.
  • Google Maps Integration - One of the standout features of the app is its integration with Google Maps. This allows users to visually explore restaurant locations and get a sense of where their orders will be delivered. React Native Maps was used to implement this feature, with markers for restaurant locations and potential delivery zones.
  • Navigation Setup - To ensure smooth navigation throughout the app, we incorporated both stack navigation for page-to-page transitions and bottom tabs for easy access to main sections. React Navigation was used to configure and manage the app’s navigation flow.
  • Styling and Responsiveness - We focused on making the app responsive across different device sizes. Using libraries like React Native Responsive Screen and Linear Gradient, we ensured the app looked great on both small and large screens, adjusting layout elements dynamically based on the device’s size.

Challenges

The main challenge in developing **ooDzee** was creating a visually complex UI without the use of backend services. While we could implement a seamless design and navigation, the absence of real-time data meant that mock data had to be used for elements like restaurant menus and order history. The next step would involve integrating APIs to provide dynamic data.

Conclusion

In conclusion, **ooDzee** is a UI-centric food ordering and delivery app built with React Native, focusing on creating an engaging user interface and smooth navigation. While it currently lacks live API integrations, the project showcases the power of React Native for creating native mobile apps and sets the stage for future enhancements, including real-time data handling, order management, and payment gateway integrations.

Lyrics Finder - Lyrics Search App

Introduction

This project is a **Lyrics Finder** app that allows users to search for song lyrics using a simple search bar. Built with **Babel** and JavaScript, the app fetches lyrics from a public API and displays them in a user-friendly format. Users can easily find lyrics by entering the song title or artist name.

Project Steps

  • Setup Babel Environment - The first step was setting up Babel to transpile ES6+ JavaScript into backward-compatible code, allowing for modern features to be used in older browsers.
  • UI/UX Design - Designed a minimal and user-friendly interface where users can input song details and receive matching lyrics quickly.
  • API Integration - Integrated a lyrics API to fetch song lyrics based on search queries.
  • Search Functionality - Implemented search functionality where users can search lyrics by song title or artist, with live results shown in real-time.

Challenges

A major challenge was dealing with asynchronous API calls and ensuring the lyrics were displayed correctly without any latency. Handling edge cases, like missing lyrics for some songs or invalid queries, also required thoughtful implementation.

Conclusion

The **Lyrics Finder** app offers a seamless experience for users to search for lyrics, powered by **Babel** and **JavaScript**. It showcases my skills in setting up modern JavaScript environments and working with APIs to deliver dynamic and interactive web applications.
